# Web DiceBot Marketplace: Seller & Buyer Guidelines

Welcome to the Web DiceBot Marketplace! To ensure a professional and transparent environment for everyone, please follow these guidelines when listing or purchasing scripts.

## 🚀 Listing Workflow (for Sellers)

- **USDT Pricing:** All scripts are priced in **USDT** to maintain value stability.
- **Mandatory Demo Video:** You must provide a link to a demo video showing the script running on a real casino with Web DiceBot for at least **30 minutes**. Link video platforms accepted: YouTube, Vimeo, Costv, Loom, TikTok, Twitter, and more.
- **Detailed Instructions:** Provide a comprehensive guide for buyers, including setup steps, strategy details, recommended balance, and important warnings.

## 🛡️ Buyer Protection (Escrow System)

To protect our community from low-quality or fraudulent scripts, we implement a **Buyer Protection** mechanism:

- **1-Hour Escrow:** When a buyer clicks "Buy Now", the transaction enters an **Escrow** state for **1 hour**.
- **Content Access:** The buyer gets immediate access to the script content to test it.
- **Reporting Issues:** If the script does not match the description or demo video, the buyer can click **"Report Issue"** within the 1-hour window. This will freeze the payment and alert the Admin team. **Note:** After reporting, the buyer must join the [Community > Marketplace](https://t.me/c/2192586959/5161) and provide evidence (screenshots/videos) for the Admin to review.
- **Dispute Resolution:**
  - If the Admin finds the report valid (**Buyer Wins**), the transaction is cancelled (Refunded), and the buyer loses access to the script.
  - If the Admin finds the script is working as intended (**Seller Wins**) or no report is made within 1 hour, the payment is automatically executed on-chain.

## 💸 Payment & Commission

- **Flexible Payments:** Buyers pay using **TRX** or **POL**. The system automatically handles the real-time conversion from your USDT price.
- **Automated Execution:** After the 1-hour Escrow window (without any report), the system executes the on-chain transfer:
  - **80%** of the payment goes directly to the **Seller's wallet**.
  - **20%** goes to the **Platform** as a commission fee.
- **Financial History:** Your "My Invoice" page will show a transparent breakdown for every successful sale:
  1. An invoice for **+100%** total revenue.
  2. An invoice for **-20%** platform commission.

## ⭐ Reputation System (Trust Rating)

Your reputation is your most valuable asset on the marketplace. It is calculated as follows:
**Score = (Upvotes × 5) - (Downvotes × 5) + (Successful Sales × 1)**

- **Trust Badge:** High reputation scores increase your listing visibility and buyer confidence.
- **Verified Voting:** Only users who have purchased the script can vote (Up or Down), ensuring 100% authentic feedback.

## ⚠️ Responsibilities & Rules

- **Sellers:** Responsible for providing working scripts and clear instructions. Malicious code will result in a **Permanent Ban**.
- **Buyers:** Responsible for testing the script within the 1-hour window. Reports made after this window cannot be processed as the funds will have already been transferred on-chain.
- **Platform:** Acts as a neutral arbiter in disputes. Admin decisions are final.
